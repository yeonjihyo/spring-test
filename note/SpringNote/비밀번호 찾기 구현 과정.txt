비밀번호 찾기 

메인화면에서 비밀번호찾기를 클릭하면 해당 uri로 넘어가서 비밀번호찾기 창이 뜬다 
아이디와 이메일확인을 통해 가입된회원정보와 일치하는지 보고 회원이면 해당 이메일로 메일을 발송한다 





선생님이 정리 

메인페이지에 비밀번호찾기 페이지로 가는 링크 추가

아이디와 이메일을 입력받을 수 있는 입력 창과 찾기버튼을 form태그안에 추가
 > 해당 form태그는 action이 /password/send

제이쿼리로 찾기버튼 클릭시 알림창 출력 테스트
 > 테스트를 성공하면 찾기버튼 클릭시 ajax를 이용하여 서버에(/password/find) 아이디와 이메일 정보를 보냄
 > 컨트롤러1 작업 수행
 > 받은 정보를 이용하여 일치하지 않으면 알림창을 띄우고(정보가 일치하지 않습니다.)
 > 일치하면 return true를 이용하여 form태그 sumit을 실행(컨트롤러2 작업 수행)
 > 일치하지 않으면 return false를 이용하여 form태그 실행하지 않음

컨트롤러 1
컨트롤러에서 해당 정보를 받기 위한 메소드가 필요하기 때문에 생성
 - RequestMapping : /password/find
 > jsp에서 전달한 매개변수 확인( id=12345678&email=stajun@naver.com)
 > &를 기준으로 id와 email을 추출 
   - id : id=12345678
   - email : email=stajun@naver.com
 > id와 email에 불필요한 xxx=을 제거 (서비스를 이용하여)
   - 서비스는 문자열을 넘겨주면 해당문자열에서 =을 기준으로 뒤에 있는 문자열만 반환하는 기능
   - id : 12345678
   - email : stajun@naver.com
 > 잘 추출햇는지 콘솔에 확인
 > 서비스에게 일을 시킴 
   - 아이디와 이메일을 알려주고, 해당 아이디와 이메일 정보가 일치하는지 확인하여 일치여부를 알려주는 기능
 > jsp에게 일치여부를 알려줌

컨트롤러2 - URI : password/send
- jsp에서 보낸 이메일이 잘 왔는지 확인
- 비밀번호 8자리 생성
- 서비스에게 이메일 주소, 제목, 내용을 알려주면서 메일을 전송하도록 함
 > 메일 전송을 위한 설정
   1. pom.xml 의존성 추가
   2. root-context.xml에 이메일 정보 설정
 > 샐플 코드를 이용하여 메일 전송
- DB에 해당 비밀번호로 회원정보 수정하도록 서비스에게 일을 시킴
